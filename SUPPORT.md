# Support

## Bugs

Use the GitHub bug report form for reproducible defects. Include the affected
implementation, package version, runtime or framework version, environment,
minimal reproduction, and relevant logs. Search existing issues first.

## Questions

Use GitHub Discussions for usage and design questions if Discussions are
enabled for the repository. Otherwise, open a clearly labeled question issue.
Questions should include a minimal example and the documentation already tried.

General implementation guides are available in the root README, package
READMEs, and [`docs/development.md`](docs/development.md).

## Security

Potential vulnerabilities are not support requests. Report them privately by
following [`SECURITY.md`](SECURITY.md). Never publish an unpatched vulnerability
in an issue or discussion.

The repository does not currently document a separate commercial or enterprise
support offering.
